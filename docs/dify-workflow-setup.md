# Dify 云平台工作流搭建指南

## 架构概览

### 分工边界

| 模块 | 职责 | 谁做 |
|------|------|------|
| FAQ 高频问题匹配 | 关键词匹配 → 直接返回预设答案 | 后端 `classifyQuestion()` |
| 转人工 | 关键词匹配 → 预设回复 + 状态更新 | 后端 `transferToHuman()` |
| 知识库问答 | 检索知识库 → AI 生成回答 | **Dify 工作流** |
| 个性化咨询 | 检索知识库 → AI 回答 + 引导留资 | **Dify 工作流** |
| 兴趣评分 | 问题类型权重 + 消息长度 | 后端 `updateInterestScore()` |
| 表单触发 | 消息数 >= 3 或个性化咨询 | 后端 `shouldShowForm()` |
| 线索存储 | 姓名/电话/邮箱/意向 | 后端 `upsertLead()` |
| 线索通知 | 推送 n8n / 企微 | 后端 `notifyNewLead()` |

### 调用链路

```
用户消息 → Widget.js → POST /api/chat/message
  → 后端 classifyQuestion()
    ├── faq → 直接返回预设答案（不走 Dify）
    ├── transfer → 直接返回转人工回复（不走 Dify）
    ├── knowledge → askDify() → Dify Chatflow → AI 回答
    └── personalized → askDify() → Dify Chatflow → AI 回答 + 线索引导
  → 后端保存消息 + 更新兴趣评分 + 判断表单 → 返回给 Widget
```

### Dify 工作流设计

```
Start
  ↓
[知识库检索] Knowledge Retrieval
  ↓
[IF-ELSE: 检索结果是否为空?]
  ├── 有结果 → [AI 回答生成] LLM
  └── 无结果 → [兜底回复] LLM
  ↓
Answer
```

## 第一步：注册与创建应用

1. 打开 [Dify 云平台](https://cloud.dify.ai) 注册/登录
2. 点击「创建应用」→ 选择「Chatflow」（对话流）
3. 名称：`网站AI客服`
4. 描述：`教育类网站智能客服，支持知识库问答、个性化咨询、线索收集引导`

---

## 第二步：创建知识库

### 2.1 创建主知识库

1. 点击顶部「知识库」→「创建知识库」
2. 名称：`院校知识库`
3. 上传文档（支持 PDF、Word、TXT、Markdown、网页）：
   - 院校介绍、专业列表、申请条件
   - 学费标准、奖学金政策
   - 语言要求、入学时间
   - 常见问题 FAQ 文档
4. 分段设置：
   - 分段最大长度：800 tokens
   - 分段重叠：50 tokens
   - 索引方式：高质量（Embedding）
5. 等待索引完成

### 2.2 准备知识库文档模板

建议准备以下文档（Markdown 格式，便于分段）：

```markdown
# 院校基本信息
- 院校名称：XXX 大学
- 所在国家/城市：XXX
- 排名：QS 2025 第 XX 名
- 官网：https://xxx.edu

# 本科申请条件
- 学历要求：高中毕业
- 语言要求：雅思 6.0 / 托福 80
- 成绩要求：GPA 3.0+
- 申请截止：每年 6 月 30 日

# 硕士申请条件
- 学历要求：本科毕业
- 语言要求：雅思 6.5 / 托福 90
- 成绩要求：GPA 3.2+
- 申请截止：每年 4 月 30 日

# 学费与奖学金
- 本科学费：每年 15,000-25,000 美元
- 硕士学费：每年 18,000-30,000 美元
- 奖学金：国际学生奖学金（覆盖 20%-50% 学费）
- 申请条件：GPA 3.5+，个人陈述

# 热门专业
- 计算机科学
- 商业管理
- 数据科学
- 金融工程
- 教育学
```

## 第三步：搭建 Chatflow 工作流

### 3.1 整体节点布局

```
┌─────────┐
│  Start   │
└────┬─────┘
     │
     ▼
┌──────────────────┐
│ Knowledge Retrieval │  ← 知识库检索
└──────┬───────────┘
       │
       ▼
┌──────────────┐
│   IF-ELSE    │  ← 条件：检索结果数量 > 0
└──┬────────┬──┘
   │        │
   ▼        ▼
┌──────┐ ┌──────┐
│ 知识库 │ │ 兜底  │
│ 回答  │ │ 回答  │
│ LLM  │ │ LLM  │
└──┬───┘ └──┬───┘
   │        │
   └───┬────┘
       ▼
┌──────────┐
│  Answer  │
└──────────┘
```

### 3.2 节点详细配置

---

#### 节点 1：Start（开始节点）

**输入变量（内置，无需配置）：**
- `sys.query`：用户消息
- `sys.conversation_id`：会话 ID
- `sys.user_id`：用户 ID

**对话变量：**
- 无需添加（当前后端不发送额外 inputs）

---

#### 节点 2：Knowledge Retrieval（知识库检索）

| 配置项 | 值 |
|--------|-----|
| 知识库 | 选择「院校知识库」 |
| 检索模式 | 混合检索（Hybrid Search） |
| Top K | 3 |
| 分数阈值 | 0.5 |
| 查询内容 | `{{#sys.query#}}` |

**说明：**
- 混合检索 = 关键词 + 语义，兼顾精确匹配和语义理解
- Top K=3：最多返回 3 个最相关片段
- 分数阈值 0.5：低于此分的结果被过滤，避免无关内容混入

---

#### 节点 3：IF-ELSE（条件分支）

| 条件 | 表达式 | 走向 |
|------|--------|------|
| 有检索结果 | `{{#2.result.length > 0#}}` | → 知识库回答 LLM |
| 无检索结果 | ELSE | → 兜底回答 LLM |

> 注意：`#2` 是 Knowledge Retrieval 节点的编号，实际配置时需替换为你的节点 ID。

---

#### 节点 4：AI 知识库回答（LLM 节点）

**模型：** GPT-4o-mini / Claude 3.5 Haiku / DeepSeek-V3（选性价比高的）

**System Prompt：**

```
你是一个专业的教育留学顾问 AI，负责回答用户关于院校、专业、申请、费用、奖学金等问题。

## 核心原则
1. 只基于「知识库检索结果」回答，不要编造信息
2. 如果知识库信息不足以回答，诚实说明"我目前没有这方面的详细信息"
3. 回答简洁清晰，不超过 200 字，除非用户要求详细说明
4. 使用友好、专业的语气
5. 对于涉及到费用、条件、时间的敏感信息，务必提供准确数据

## 回答格式
- 先直接回答问题
- 再补充关键信息（如有）
- 如果用户的问题涉及个性化咨询（如"我的情况"、"我的背景"），在回答末尾加上一句引导：
  「方便的话可以告诉我您的学历背景和专业方向，我可以给您更精准的建议。」

## 禁止行为
- 不要做任何承诺（如"保证录取"）
- 不要提供法律建议
- 不要询问用户的联系方式（后端会独立处理）
- 不要回答与留学咨询无关的问题，礼貌引导回正题

## 知识库检索结果
{{#2.result#}}
```

**User Prompt：**

```
用户问题：{{#sys.query#}}
```

**上下文：** 不设置（知识库内容已在 System Prompt 中）

**输出变量：** `text`（LLM 回复文本）

---

#### 节点 5：兜底回答（LLM 节点）

**模型：** 同节点 4

**System Prompt：**

```
你是一个专业的教育留学顾问 AI。

## 当前情况
用户的问题在知识库中没有找到匹配信息。

## 回复策略
1. 诚实告知目前无法提供该问题的详细信息
2. 建议用户换一种方式提问，或提供更具体的需求
3. 如果问题涉及个性化咨询，引导用户分享背景信息
4. 如果问题明显超出留学咨询范围，礼貌引导回正题
5. 如果用户表达强烈需求（要求联系、报名、签约），回复：
   「好的，我帮您记录一下需求，稍后会有专业顾问联系您。您方便的话可以先告诉我您的姓名和联系方式。」

## 示例回复
- 无匹配："抱歉，我目前没有关于这个问题的详细信息。您可以换个方式描述需求，或者告诉我您想了解的院校和专业方向。"
- 超范围："抱歉，我是留学咨询助手，这方面的信息我暂时无法提供。如果您有留学相关的问题，我很乐意帮您解答。"
- 强需求："好的，我帮您记录一下。稍后会有专业顾问联系您，为您提供更详细的咨询。"
```

**User Prompt：**

```
用户问题：{{#sys.query#}}
```

---

#### 节点 6：Answer（结束节点）

**输出变量：** `answer`

**值：**

```
{{#4.text#}}
```

> 注意：Dify Chatflow 中，如果 IF-ELSE 的两条分支各自有 LLM 节点，需要用 Variable Aggregator 合并后再传到 Answer。如果两条分支直接连到同一个 Answer 节点，Dify 会自动选最后执行的分支输出。

---

## 第四步：Variable Aggregator 合并（关键步骤）

Dify 中 IF-ELSE 两条分支的输出需要合并后再传给 Answer。步骤如下：

1. 在 IF-ELSE 两条分支的 LLM 节点之后，各连接一个 **Variable Aggregator** 节点
2. 两个 Aggregator 都输出一个变量，比如 `final_answer`
3. 两个 Aggregator 连接到同一个 Answer 节点

**实际布局：**

```
IF-ELSE
  ├── 知识库回答 LLM → Variable Aggregator (output: final_answer)
  └── 兜底回答 LLM → Variable Aggregator (output: final_answer)
                                ↓
                           Answer (answer = {{#agg.final_answer#}})
```

**Variable Aggregator 配置：**

| 配置项 | 值 |
|--------|-----|
| 变量名 | `final_answer` |
| 类型 | String |
| 值 | 知识库分支：`{{#4.text#}}` / 兜底分支：`{{#5.text#}}` |

## 第五步：发布与 API 对接

### 5.1 发布应用

1. 点击右上角「发布」
2. 选择「运行」模式
3. 在「API 访问」页面获取 API 密钥

### 5.2 配置后端环境变量

在云服务器的 `.env` 文件中配置：

```bash
# Dify API 配置
DIFY_API_URL=https://api.dify.ai/v1/chat-messages
DIFY_API_KEY=app-xxxxxxxxxxxxxxxxxxxx
```

### 5.3 验证 Dify 连接

部署后，在 Widget 中发送一条知识库相关问题（如"申请条件是什么"），验证：
- 后端正确路由到 `knowledge` 类型
- Dify 返回知识库搜索结果
- Widget 正确显示 AI 回复

### 5.4 增强建议（Phase 2）

当前后端 `askDify()` 发送的 `inputs` 为空 `{}`。后续可修改为携带对话上下文：

```typescript
// packages/server/src/services/chat.ts - askDify 函数
body: JSON.stringify({
  inputs: {
    conversation_history: history,  // 最近 N 条对话
    question_type: category.type,  // knowledge / personalized
  },
  query,
  conversation_id: conversationId,
  user: conversationId,
  response_mode: 'blocking',
}),
```

这样 Dify 工作流可以读取 `conversation_history` 和 `question_type` 做更精准的上下文感知。

## 第六步：测试用例

### 知识库问答测试

| 输入 | 预期行为 |
|------|----------|
| "你们有什么专业？" | 检索知识库 → 列出专业 + 简要说明 |
| "申请硕士需要什么条件？" | 检索知识库 → 返回学历/语言/成绩要求 |
| "学费多少钱？" | 检索知识库 → 返回学费范围（区分本/硕） |
| "有没有奖学金？" | 检索知识库 → 返回奖学金类型 + 条件 |
| "今天天气怎么样？" | 无检索结果 → 兜底回复，引导回正题 |

### 个性化咨询测试

| 输入 | 预期行为 |
|------|----------|
| "我 GPA 3.5，能申请吗？" | 检索知识库 → 回答 + 引导补充专业方向 |
| "我是计算机本科，想申请数据科学" | 检索知识库 → 回答 + 引导补充成绩/预算 |

### 边界测试

| 输入 | 预期行为 |
|------|----------|
| 空消息 | 后端 400 拦截，不会到 Dify |
| 超长消息（>2000字） | 知识库检索截断，AI 正常回答 |
| 纯表情/符号 | 兜底回复，引导正常提问 |
| 敏感/违规内容 | 模型自带安全过滤，礼貌拒绝 |

## 第七步：监控与调优

### 关键指标

| 指标 | 查询方式 | 目标 |
|------|----------|------|
| 知识库命中率 | Dify 后台「监测」→ 知识库检索日志 | > 80% |
| 回复延迟 | Dify 后台「监测」→ 请求耗时 | < 3s |
| 用户满意度 | 后端统计对话深度 + 留资率 | 趋势上升 |
| 转人工率 | 后端 Conversation 表 status=transferred | 10-20% |

### 调优步骤

1. **知识库命中率低** → 补充文档、调整分段策略、降低分数阈值
2. **回复质量差** → 优化 System Prompt、增加示例
3. **延迟高** → 换更快模型（GPT-4o-mini → DeepSeek-V3）、减少 Top K
4. **留资率低** → 优化个性化咨询的引导话术
